# Code Of Conduct

## Our Standard

This project aims to be practical, respectful, and welcoming to people who want to use, test, or improve it.

Examples of expected behavior:

- be respectful and direct
- assume good intent before escalating
- focus criticism on the code or idea, not the person
- share context when reporting bugs or requesting changes
- help keep the project useful for newcomers

Examples of unacceptable behavior:

- harassment, insults, or personal attacks
- discriminatory language or exclusionary behavior
- deliberately disruptive issue, PR, or discussion behavior
- publishing private tokens, private network details, or sensitive data from other users
- spam or repeated bad-faith requests

## Scope

This applies to issues, pull requests, release discussions, and other project spaces on GitHub.

## Enforcement

Project maintainers may edit, hide, lock, or remove content that violates this code of conduct. Repeated or severe violations may lead to a ban from participation.

If you experience abuse, use GitHub's reporting tools where appropriate and contact the maintainer through GitHub.
